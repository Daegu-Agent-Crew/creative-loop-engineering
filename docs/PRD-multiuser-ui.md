# PRD: 창의적 루프 엔지니어링 시스템 — 다중 사용자 UI

> 버전: 1.0 | 작성: 2026-06-17 | 상태: Draft

---

## 1. 프로젝트 개요

### 1.1 배경
Addy Osmani의 Loop Engineering 6가지 부품 + 기존 랄프톤 생태계 Phase Split 구조를 통합한 **창의적 루프 엔지니어링 시스템**의 다중 사용자용 UI를 설계합니다.

### 1.2 목표
- **"선택하면 완성된다" 경험** 제공
- 비기술자도 3단계 카드 선택으로 창의적 산출물 생성
- 다중 사용자가 동시에 프로젝트를 진행하고 결과를 공유

### 1.3 대상 사용자
- 1차: 대구 에이전트 빌더 커뮤니티 멤버 (10+명)
- 2차: 일반 창작자 (만화, 일러스트, 글쓰기)
- 3차: 팀 기반 콘텐츠 스튜디오

---

## 2. 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **선택 우선** | 텍스트 입력 최소화, 카드/슬라이더/버튼으로 선택 |
| **진행 가시화** | 5-Phase 파이프라인이 시각적으로 보여야 함 |
| **진입 장벽 제로** | 가입 없이 체험 가능, 튜토리얼 3장 슬라이드 |
| **소셜 구성** | 다른 사람 작업 구경, 템플릿 복제, 피드백 공유 |
| **모바일 동등** | 카드 선택 + 승인만 되면 모바일에서도 충분 |

## 3. 페이지 구조

```
├── /                    → 랜딩 + 체험 (게스트 가능)
├── /dashboard           → 내 프로젝트 목록 + 진행 상태
├── /new                 → 새 프로젝트 생성 (3단계 카드)
├── /project/:id         → 프로젝트 진행 대시보드
│   ├── Phase 진행 바
│   ├── 중간 결과물 썸네일
│   ├── 피드백/승인 UI
│   └── 최종 결과물 갤러리
├── /gallery             → 공개 갤러리 (모든 사용자 결과물)
├── /templates           → 템플릿 마켓 (복제 후 커스텀)
└── /settings            → 계정/환경설정
```

## 4. 상세 스펙

### 4.1 랜딩 페이지 (`/`)

**목표:** 30초 안에 "이게 뭔지 + 해보고 싶다" 느끼게

- Hero: 애니메이션 루프 비주얼 (Phase들이 순환하는 SVG 애니메이션)
- "바로 체험하기" CTA → 가입 없이 바로 /new로 이동
- 갤러리에서 최근 완성 작품 3~4개 썸네일
- 카테고리 태그: 만화 / 일러스트 / 스토리보드 / 글쓰기

### 4.2 새 프로젝트 생성 (`/new`)

**핵심: 3단계 카드 선택 UI**

```
┌──────────────────────────────────────────────┐
│  Step 1/3: 주제 선택                          │
│                                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐        │
│  │ 🚀 SF   │ │ 🧙 판타지│ │ 🏙️ 현대 │        │
│  │ 미래     │ │ 마법     │ │ 일상     │        │
│  └─────────┘ └─────────┘ └─────────┘        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐        │
│  │ 📜 역사 │ │ 🎭 드라마│ │ ✏️ 커스텀│        │
│  └─────────┘ └─────────┘ └─────────┘        │
│                                              │
│          선택 시 오른쪽에 실시간 프리뷰         │
│          (해당 주제의 참고 이미지/팔레트)       │
│                                              │
│  ──────────────────────────────────────────  │
│  Step 2/3: 화풍     Step 3/3: 분위기          │
│                                              │
│            [ 🚀 생성 시작 ]                    │
└──────────────────────────────────────────────┘
```

- **카드 선택:** 클릭 한 번, 다중 선택 가능 (예: SF + 역사 혼합)
- **커스텀 카드:** 텍스트로 자유 입력 (고급 사용자용)
- **실시간 프리뷰:** 선택한 조합의 예시 결과물이 썸네일로 표시
- **"AI가 추천해줘" 버튼:** 랜덤/트렌딩 조합 자동 선택

### 4.3 프로젝트 진행 대시보드 (`/project/:id`)

**핵심: 파이프라인이 보이는 인터페이스**

```
┌──────────────────────────────────────────────┐
│  프로젝트: SF 수채화 만화                     │
│  ──────────────────────────────────────────  │
│                                              │
│  ① Define ───→ ② Generate ───→ ③ Evaluate   │
│      ✅              🔄               ⏳     │
│                                         │    │
│  ④ Refine ←──── ⑤ Deliver ←────────────┘    │
│      ⏳              ⏳                     │
│                                              │
│  ──────────────────────────────────────────  │
│  📋 현재 단계: Generate (2/5)                │
│                                              │
│  생성 중인 결과물:                            │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐               │
│  │ 🖼️ │ │ 🖼️ │ │ ⏳ │ │ ⏳ │               │
│  │ #1 │ │ #2 │ │ #3 │ │ #4 │               │
│  └────┘ └────┘ └────┘ └────┘               │
│                                              │
│  ──────────────────────────────────────────  │
│  피드백 (게이트 #2):                         │
│                                              │
│  방향성:   ★★★★☆ [슬라이더]                  │
│  품질:     ★★★★★ [슬라이더]                  │
│                                              │
│  [ ✅ 승인 ]  [ 🔄 재생성 ]  [ ✏️ 방향 수정 ] │
│  [ 💬 코멘트 (선택) ___________ ]            │
└──────────────────────────────────────────────┘
```

**게이트(피드백) UI 규칙:**
- 별점/슬라이더가 기본 (텍스트 입력은 선택사항)
- "이 방향이 좋아요" 원클릭 저장 → 다음 생성에 자동 반영
- 재생성 시 이전 결과물과 비교 뷰 제공
- 게이트 통과 시 자동 다음 Phase로 진행 (알림)

### 4.4 공개 갤러리 (`/gallery`)

```
┌──────────────────────────────────────────────┐
│  갤러리                    🔍 검색  🏷️ 필터   │
│                                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│  │  썸네일   │ │  썸네일   │ │  썸네일   │     │
│  │          │ │          │ │          │      │
│  │ SF수채화 │ │ 현대팝아트│ │ 판타지수채│     │
│  │ by @user │ │ by @user │ │ by @user │      │
│  │ ❤️ 42    │ │ ❤️ 28    │ │ ❤️ 35    │      │
│  │ [복제]   │ │ [복제]   │ │ [복제]   │      │
│  └──────────┘ └──────────┘ └──────────┘      │
└──────────────────────────────────────────────┘
```

- 무한 스크롤 + Masonry 그리드
- 좋아요, 복제(템플릿으로 가져오기), 공유
- 필터: 카테고리, 화풍, 최신, 인기
- 클릭 → 전체 결과물 보기 + 제작 과정 타임라인

### 4.5 템플릿 마켓 (`/templates`)

- 완성된 프로젝트의 설정값을 템플릿으로 저장
- 한 번 클릭으로 내 프로젝트로 복제
- 카테고리별 정렬, 다운로드 수 랭킹
- 커뮤니티 템플릿 + 공식 추천 템플릿

## 5. 사용자 플로우

### 5.1 신규 사용자 (First Experience)

```
랜딩 → "체험하기" 클릭
  → 주제 카드 선택 (1클릭)
  → 화풍 카드 선택 (1클릭)
  → 분위기 카드 선택 (1클릭)
  → "생성 시작" 클릭
  → 진행 대시보드에서 자동 진행 관찰
  → 게이트에서 "승인" 클릭 (또는 별점)
  → 완성! 결과물 갤러리에 자동 공개
  → "계정 만들고 저장하기" 제안
```

**목표 시간:** 첫 결과물까지 5분 이내

### 5.2 숙련 사용자

```
대시보드 → 진행 중 프로젝트 확인
  → 게이트에서 세부 피드백
  → 여러 프로젝트 병렬 진행
  → 템플릿으로 빠른 시작
  → 갤러리에서 영감 수집
```

## 6. 기술 아키텍처

### 6.1 프론트엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Vanilla HTML/CSS/JS (SPA) | GitHub Pages 배포, 의존성 최소 |
| 스타일 | CSS Variables + Grid/Flex | 기존 index.html과 일관 |
| 상태관리 | localStorage + sessionStorage | 서버 없이 동작 |
| 라우팅 | hash-based SPA router | GitHub Pages 호환 |

### 6.2 백엔드 (Phase 2)

| 항목 | 선택 | 이유 |
|------|------|------|
| API | GitHub Actions + API | 무서버 백엔드 |
| 저장소 | GitHub Issues + Discussions | 템플릿/피드백/댓글 |
| 사용자 | GitHub OAuth | 커뮤니티에 자연스러움 |
| 이미지 | GitHub Releases / imgur | 결과물 저장 |

### 6.3 배포

- Phase 1: 정적 GitHub Pages (클라이언트 전용)
- Phase 2: Cloudflare Workers API 추가

## 8. 디자인 시스템

### 8.1 색상 팔레트 (기존 시스템과 일관)

```css
--bg: #0a0e17;
--surface: #111827;
--surface2: #1a2235;
--border: #1e293b;
--text: #e2e8f0;
--text2: #94a3b8;
--accent: #6366f1;
--accent2: #818cf8;
--green: #22c55e;
--amber: #f59e0b;
--red: #ef4444;
```

### 8.2 카드 디자인

- 라운드 12px, border 1px, hover 시 accent로 변환
- 카드 내부: 아이콘 + 타이틀 + 한 줄 설명
- 선택 상태: accent border + glow 효과

### 8.3 모바일 레이아웃

- 카드: 2열 그리드 → 모바일 1열
- 진행 바: 가로 스크롤 또는 세로 스태킹
- 피드백: 풀스크린 모달

|------|------|-----------|
