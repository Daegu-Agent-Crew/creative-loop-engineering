# OpenClaw Visualizer Phase 1 - 구조 + 로직

RPG 던전 맵 스타일로 OpenClaw 4페이즈 아키텍처를 시각화하는 웹 앱.
Phase 1에서는 애니메이션 없이 **구조와 로직만** 구현합니다.

---

## 📋 구현할 것

### 1. 4층 던전 맵 레이아웃

```
┌──────────────────────────────────────┐
│  🎮 OpenClaw Visualizer              │
│  [시뮬레이션 모드] [라이브 모드]      │
├──────────────────────────────────────┤
│  ① INGESTION 층 (초록색 테두리)      │
│  ┌────┐  ┌────┐  ┌────┐             │
│  │ TG │  │ DC │  │ WA │  채널 어댑터 │
│  └──┬─┘  └──┬─┘  └──┬─┘             │
│     └───────┼────────┘               │
├─────────────▼────────────────────────┤
│  ② CONTROL PLANE 층 (주황색 테두리)  │
```

### 2. 캐릭터 상태창 (우측 사이드바)
```

│ ⏱️ 레이턴시: 1.2s │
│ XP: 1,234/2000   │
│ [=====>    ] 62% │
└─────────────────┘
```

### 3. 퀘스트 로그 (우측 사이드바 하단)
메시지 처리 과정을 단계별로 텍스트로 기록:
```
📜 퀘스트 로그:
  00:01 [①] 텔레그램 수신 "날씨 알려줘"
  00:02 [②] 라우팅 → 메인 세션
  00:03 [③] Pi Runtime 추론...
  00:04 [④] web_search("날씨 서울")
  00:05 [③] 응답 생성...
  00:06 [①] 텔레그램 응답 전송
  ✅ XP +50 획득
```

### 4. 미니맵 (좌측 상단)
4층 구조를 축소해서 보여주고, 현재 활성 층을 하이라이트.

### 5. 시뮬레이션 모드 (3개 시나리오)

#### 시나리오 1: "날씨 알려줘" (초보)
```
단계:
1. ① TG 노드 활성화 → "수신"
2. ② Router 노드 활성화 → "라우팅"
3. ③ Pi Runtime 노드 활성화 → "추론 중..."
4. ④ Search 노드 활성화 → "web_search 실행"
5. ③ Pi Runtime → "응답 생성"
6. ① TG 노드 → "응답 전송"
7. 완료! XP +50
```

#### 시나리오 2: "랄프톤으로 게임 만들어줘" (보스)
```
단계:
1. ① TG 수신
2. ② 라우팅
3. ③ Pi Runtime 추론 (길게)
4. ④ File 노드 → read(PRD.md)
5. ④ File 노드 → read(v4/index.html)
6. ④ Exec 노드 → claude 실행 (길게 대기)
7. ④ File 노드 → 결과 확인
8. ④ Message 노드 → send(telegram)
9. ③ 응답 생성
10. 완료! XP +500, 레벨업!
```

#### 시나리오 3: "리마인더 설정해줘" (시간)
```
단계:
1. ① TG 수신
2. ② 라우팅
3. ③ 추론
4. ④ Memory 노드 → memory_search
5. ② Cron 노드 → cron(add)
6. 5초 대기
7. ② Cron 트리거!
8. ③ systemEvent 처리
9. ① 알림 전송
10. 완료!
```

### 6. 컨트롤 패널
- 시나리오 선택 드롭다운 (초보/보스/시간)
- "▶ 시작" 버튼
- "⏭️ 다음 단계" 버튼 (수동 진행)
- "🔄 리셋" 버튼
- 속도 조절 (0.5x, 1x, 2x)

### 7. 라이브 모드 (기본 숨김)
- WebSocket 연결 버튼 (ws://127.0.0.1:18800/ws)
- OpenAI API 연결 (http://127.0.0.1:18789/v1/chat/completions)
- 연결 상태 표시
- 메시지 입력창
