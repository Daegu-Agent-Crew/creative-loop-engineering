📖 AI 설명서
Q: AI는 어떻게 "생각"하나요?▼
AI는 4단계로 생각합니다:

1. 수신 - 사용자 메시지를 받습니다
2. 이해 - 무엇을 원하는지 파악합니다
3. 도구 - 필요한 도구를 선택하고 실행합니다
4. 응답 - 결과를 바탕으로 답변을 생성합니다

이 과정을 "메시지 여행" 탭에서 직접 눈으로 확인할 수 있습니다!
Q: 왜 같은 질문에 다른 답이 나오나요?▼
AI는 확률적으로 다음 단어를 예측합니다. 온도(temperature) 설정에 따라 창의성과 일관성의 균형이 달라집니다.

- 낮은 온도 (0.1~0.3): 일관되고 예측 가능한 답변
- 높은 온도 (0.7~1.0): 창의적이고 다양한 답변

마치 주사위를 던지는 것처럼, 매번 약간씩 다른 결과가 나옵니다.
Q: AI의 기억은 어떻게 작동하나요?▼
두 가지 기억이 있습니다:

💭 단기 기억 (Context Window)
현재 대화 내용을 기억합니다. 대화가 길어지면 오래된 내용은 잊혀집니다. 보통 128K~1M 토큰의 용량이 있습니다.

💾 장기 기억 (MEMORY.md)
중요한 정보를 파일에 저장합니다. 다음 대화에서도 기억할 수 있습니다. 사용자의 선호도, 프로젝트 정보 등을 저장합니다.
Q: 툴(Tool)이 뭔가요?▼
AI가 직접 행동할 수 있는 도구입니다:

🔍 검색 - 웹에서 실시간 정보 검색
📁 파일 읽기/쓰기 - 파일 시스템 조작
⚙️ 코드 실행 - 프로그램 실행
💾 메모리 저장 - 중요 정보 기록
📨 메시지 전송 - 텔레그램 등으로 응답
👁️ 비전 - 이미지 분석

AI는 질문의 성격에 따라 적절한 툴을 자동으로 선택합니다.
Q: 토큰(Token)이 뭔가요?▼
토큰은 AI가 텍스트를 처리하는 최소 단위입니다.

- 영어: 약 4글자 = 1토큰 ("hello" = 1토큰)
- 한국어: 약 1~2글자 = 1토큰 ("안녕" = 2토큰)

토큰이 중요한 이유:
💰 비용: 토큰 수에 따라 API 비용이 결정됩니다
📏 한계: Context Window 크기가 토큰으로 제한됩니다
⏱️ 속도: 토큰이 많을수록 응답이 느려집니다
Q: AI 에이전트와 챗봇의 차이는?▼
챗봇: 대화만 가능. 텍스트를 주고받는 것이 전부입니다.

AI 에이전트: 대화 + 행동이 가능합니다.
- 스스로 도구를 선택하고 실행
- 여러 단계의 작업을 자율적으로 수행
- 외부 시스템과 연동 (검색, 파일, API 등)
- 결과를 분석하고 다음 행동을 결정

OpenClaw는 AI 에이전트입니다!
Q: 컨텍스트 윈도우(Context Window)란?▼
AI가 한 번에 "볼 수 있는" 텍스트의 양입니다.

마치 책상 위 공간과 같습니다:
- 책상이 넓으면(큰 컨텍스트) 더 많은 자료를 동시에 볼 수 있고
- 책상이 좁으면(작은 컨텍스트) 오래된 자료를 치워야 합니다

모델별 크기:
- GPT-4: 128K 토큰
- Claude: 200K ~ 1M 토큰

"AI 지갑" 바에서 현재 사용률을 확인할 수 있습니다.
Q: AI는 감정이 있나요?▼
AI는 감정을 시뮬레이션할 수 있지만, 실제로 느끼지는 않습니다.

"슬프다"고 말할 수 있지만, 그것은 학습된 패턴에 따른 텍스트 생성일 뿐입니다. 하지만 사용자와의 상호작용에서 적절한 공감을 표현하도록 설계되어 있습니다.

OpenClaw의 SOUL.md 파일에는 AI의 "성격"이 정의되어 있어, 일관된 페르소나를 유지합니다.
Q: 비용은 어떻게 계산되나요?▼
AI API 비용은 주로 토큰 수로 결정됩니다:

입력 토큰: 사용자의 메시지 + 시스템 프롬프트 + 대화 기록
출력 토큰: AI의 응답

예시 (Claude Sonnet 기준):
- 간단한 질문: ~$0.002 (2원)
- 복잡한 코딩: ~$0.05 (50원)
- 대규모 프로젝트: ~$0.50+ (500원+)

"AI 지갑"에서 실시간으로 비용을 확인할 수 있습니다.
Q: "랄프톤"이 뭔가요?▼
랄프톤(Ralphton)은 OpenClaw의 AI 코딩 에이전트입니다.

사용자가 "게임 만들어줘"처럼 복잡한 요청을 하면, Pi Runtime이 랄프톤을 호출합니다. 랄프톤은:

1. 요구사항 문서(PRD)를 읽고
2. 기존 코드를 분석하고
3. Claude Code를 실행하여 코드를 생성하고
4. 결과를 검증합니다

"보스" 시나리오에서 이 과정을 볼 수 있습니다!
